This example is a placeholder for a fit of an NSNL (2S2L) event, OGLE-2016-BLG-1003, published here:
https://ui.adsabs.harvard.edu/abs/2017ApJ...841...75J/abstract

Data downloaded from the paper is dbf1.txt. convert_data.py generates the *.dat EXOZIPPy input files.

ob161003.yaml is seeded with the 2S2L standard solution in Table 1 of the above paper.

Currently, the structure of the parameter file is incorrect and the plumbing is incomplete. 

I want to see if Fable 5 can take this as is and make NSNL fits work and recover the published values for this system.
